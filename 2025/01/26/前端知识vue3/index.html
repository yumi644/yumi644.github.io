2.本地写好的vue项目通过nvm，npm,nodejs生成dist文件的流程##

在命令行中使用 cd 命令导航到你的 Vue 项目目录，例如：

>cd "E:\python code\前端\POEMON-BLOG-main\My-Blog-FrontOpen"

确保在项目目录中执行以下命令，以安装项目所需的依赖：

>npm install

现在，你可以运行构建命令来生成 dist 文件夹。通常，这个命令在 package.json 的 scripts 部分中定义，通常是 build。执行以下命令：

>npm run build

部署 dist 文件

接下来，你可以将 dist 文件夹中的内容部署到服务器或静态网站托管服务，如 Netlify、Vercel、GitHub Pages 等。

3. nodejs，npm,nvm的概念##

Node.js 是服务器端的 JavaScript 运行时环境。
npm 是 Node.js 的包管理工具，用于管理依赖和发布包。
nvm 是用于管理 Node.js 版本的工具，方便开发者在不同版本之间切换。

4. Webpack概念##

Webpack 是一个现代 JavaScript 应用程序的静态模块打包工具。它主要用于将复杂的前端应用程序打包成更小的、可管理的文件，从而优化加载性能。旨在解决现代前端开发中的模块化和资源管理问题，使得开发者能够构建高效、可维护的应用程序。

8. CDN（内容分发网络）##

CDN（内容分发网络）是一种通过全球分布的服务器加速内容传输，提高网站加载速度和可靠性的方法

使用 CDN
在网页中使用 CDN 资源非常简单，只需在 HTML 文件中添加相应的链接。例如，要使用 Bootstrap 的 CSS：

link rel=”stylesheet” href=”https://stackpath.bootstrapcdn.com/bootstrap/4.5.2/css/bootstrap.min“

这段代码会从 Bootstrap 的 CDN 服务器加载 CSS 文件，而不是从您的服务器加载。这样可以提高加载速度，并减轻您服务器的压力。

15.生命周期函数

   组件实例从创建到销毁过程中不同时间点自动调用的函数

(1)挂载阶段

   onBeforeMount

           在组件实例即将被挂载到DOM树之前调用

           此时模板还未编译或渲染到DOM,通常用于执行初始化操作,

           如:获取异步数据、设置初始属性值等

   onMounted

           在组件成功挂载到DOM并完成首次渲染后调用

           此时可以访问和操作DOM元素,

           并执行与页面交互相关的逻辑

(2)更新阶段

   onBeforeUpdate (由于响应式数据变化)

           在组件更新之前即将重新渲染时调用

           可以根据新的参数判断是否需要进行特殊处理,

           甚至可以选择阻止此次更新过程

   onUpdated

           在组件完成更新并重新渲染后调用

           可以基于新的渲染结果处理更新后的数据

(3)卸载阶段

   onBeforeUnmount

           在组件从DOM中销毁之前调用

           用于释放资源,如:清理计时器、解绑事件监听器等

   onUnmounted

           在组件已经从DOM中移除并销毁后调用

           确保组件所占用的所有资源都被正确释放

(4)错误处理

   onErrorCaptured

           在捕获到组件中的错误时调用

           用于处理错误,如:记录错误日志等

总结：组件挂载的过程

模板编译

   将组件的模板转换为JS代码

渲染

   在模板编译后生成的JS代码渲染到页面上,

   生成虚拟DOM

挂载

   在渲染完成后将虚拟DOM挂载到真实的DOM树上,

   使其在页面上显示出来

18.pinia插件persistedstate持久化存储使用

问：pinia持久化插件也是存储到localStorage中,为什么不直接使用localStorage?

（1）自动状态同步

   持久化插件自动将Pinia的状态存储到localStorage中,

   无需手动处理状态的读取和写入

（2）易用性

   无需手动处理localStorage的键值对存储、数据转换等繁琐过程

（3）与Vue组件状态紧密集成

   持久化插件与Vue组件的响应式数据完美结合

   当状态改变时,依赖这些状态的组件会自动更新视图

   与仅仅从localStorage中读取静态数据相比更加灵活和强大

main.js

	//导入Pinia的createPinia方法,用于创建Pinia实例(状态管理库)
	import { createPinia } from 'pinia'
	//从 pinia-plugin-persistedstate 模块中导入 piniaPluginPersistedstate
	import piniaPluginPersistedstate from 'pinia-plugin-persistedstate'

	import App from './App.vue'

	//创建一个Pinia实例,用于在应用中集中管理状态(store)
	const pinia = createPinia()

	//将插件添加到 pinia 实例上
	pinia.use(piniaPluginPersistedstate)
	
	//createApp(App).mount('#app')
	const app = createApp(App)
	app.use(pinia) //将Pinia实例注册到Vue应用中
	app.mount('#app')

web.js

	import { reactive, ref } from 'vue'
	import { defineStore } from 'pinia'
	
	/*
	  定义一个基于 Pinia 的 Store
	  第1个参数 web 是 useWebStore 在应用中的唯一标识符(ID)
	  第2个参数是 Setup函数 或 Option对象
	*/
	export const useWebStore = defineStore('web', () => {
	  //定义一个响应式对象，存储网站信息
	  const web = reactive({
	    title: "邓瑞编程",
	    url: "dengruicode.com"
	  })
	
	  //定义一个响应式引用，存储用户数
	  const users = ref(1000)
	  
	  //定义方法
	  const userAdd = () => {
	    users.value++
	  }
	
	  return {
	    web,
	    users,
	    userAdd
	  }
	},
	{
	  //持久化存储到 localStorage 中
	  persist: true
	})

19.App.vue 是 Vue 应用的根组件，主要负责以下几个方面：

  1. 根组件
    App.vue 是整个 Vue 应用的入口组件，它是所有其他组件的父组件。所有子组件都将嵌套在这个组件中。

  2. 结构和布局
    在 App.vue 中，你可以定义应用的基本结构和布局，例如导航栏、侧边栏、底部信息等。这些内容通常在所有页面中共享。

  3. 路由视图
    如果使用了 Vue Router，App.vue 通常包含一个 组件，用于渲染匹配的子路由组件。这使得应用能够根据路由动态加载不同的视图。

  4. 状态管理
    在 App.vue 中，可以使用状态管理工具（如 Pinia 或 Vuex）来管理全局状态，并将其传递给子组件。

20.main.js 是 Vue 应用的入口文件，主要负责以下几个关键任务：

main.js 是 Vue 应用的入口文件，主要负责以下几个关键任务：

  1. 创建 Vue 应用实例
    main.js 中使用 createApp 函数创建一个 Vue 应用实例。这是应用的起点，所有的配置和组件都会从这里开始。

  2. 注册全局组件和插件
    在 main.js 中，你可以注册全局的组件、指令或插件。这些组件和功能可以在应用的任何地方使用。例如，使用 Vue Router 或 Pinia 时，会在这里进行配置和注册。

  3. 配置状态管理
    如果使用了状态管理工具（如 Pinia 或 Vuex），在 main.js 中通常会创建相应的实例并将其注册到应用中。

  4. 挂载组件
    最后，main.js 中会将创建的 Vue 应用实例挂载到一个 DOM 元素上，通常是 index.html 文件中的一个 div，例如 #app。

     // main.js
 import { createApp } from 'vue';
 import App from './App.vue'; // 导入根组件
 import { createPinia } from 'pinia'; // 导入 Pinia 插件
 
 const app = createApp(App); // 创建 Vue 应用实例
 const pinia = createPinia(); // 创建 Pinia 实例
 
 app.use(pinia); // 注册 Pinia 插件
 app.mount('#app'); // 挂载应用到 DOM

21.app.mount(‘#app’);

app.mount(‘#app’); 是 Vue 3 中用于挂载 Vue 应用的命令。这行代码的含义和作用如下：

含义
挂载：将 Vue 应用实例与 DOM 中的某个元素关联起来，使得 Vue 能够在该元素内渲染组件和内容。
作用
选择器：’#app’ 是一个 CSS 选择器，表示要挂载的 DOM 元素的 ID。在 HTML 文件中，通常会有一个

作为 Vue 应用的挂载点。
启动渲染：调用 mount 方法后，Vue 将开始渲染根组件（通常是 App.vue）及其子组件，并将生成的内容插入到指定的 DOM 元素中。
响应式更新：一旦挂载，Vue 会开始管理该 DOM 元素内的内容，并根据数据的变化自动更新视图。

23.vue2中的选项式API和vue3组合式API的区别

data() 方法
用途：用于定义组件的响应式数据。
返回值：返回一个对象，包含组件的状态或属性。
特点：
该对象中的属性可以在模板中使用，并且当这些属性的值改变时，视图会自动更新。
data() 是一个函数，因为它可以在每次组件实例化时返回一个新的对象，从而确保每个组件实例都有自己的数据副本。
methods 中的方法
用途：用于定义组件内可调用的函数或逻辑。
特点：
方法可以被模板中的事件（如点击）调用，或者在其他方法中引用。
在方法中，可以通过 this 访问组件的 data、computed 属性以及其他方法。
方法通常用于处理用户交互、执行逻辑计算或更新数据等。

（1） 组织方式

a.选项式 API（Vue 2）：

通过将组件的不同部分（如 data、methods、computed、watch 等）分开组织在一个对象中来定义组件。
这种方式在小型组件中非常直观，但在大型组件中可能会导致逻辑分散，难以管理。

	export default {
	  data() {
	    return {
	      message: 'Hello, Vue!',
	    };
	  },
	  methods: {
	    greet() {
	      alert(this.message);
	    },
	  },
	  computed: {
	    reversedMessage() {
	      return this.message.split('').reverse().join('');
	    },
	  },
	};

b.组合式 API（Vue 3）：

使用 setup() 函数来组织组件逻辑，使得功能更容易组合和复用。可以在 setup() 中定义响应式状态、方法和计算属性。
这种方式使得逻辑更加集中，适合复杂组件和大型应用。

	import { ref, computed } from 'vue';
	
	export default {
	  setup() {
	    const message = ref('Hello, Vue!');
	    
	    const greet = () => {
	      alert(message.value);
	    };
	
	    const reversedMessage = computed(() => {
	      return message.value.split('').reverse().join('');
	    });
	
	    return { message, greet, reversedMessage };
	  },
	};

（2）响应式系统

a.选项式 API：

数据是通过 data() 方法定义的。Vue 会自动将返回的对象转换为响应式。

b.组合式 API：

使用 ref 或 reactive 创建响应式数据。ref 用于基本数据类型，而 reactive 用于对象。

（3）逻辑复用

a.选项式 API：

逻辑复用通常依赖于 mixins 和继承，可能导致命名冲突和复杂性。

b.组合式 API：

通过组合函数（Composition Functions）实现逻辑复用，更加灵活和清晰。可以将相关逻辑封装成单独的函数，便于在多个组件中复用。

（4）类型支持

a.选项式 API：

对 TypeScript 的支持较弱，类型推导不够灵活。

b.组合式 API：

提供更好的 TypeScript 支持，类型推导更为强大和灵活，适合大型项目。

（5）生命周期钩子

a.选项式 API：

生命周期钩子是组件选项的一部分，如 mounted()、created() 等。

b.组合式 API：

在 setup() 中使用生命周期钩子函数，如 onMounted、onCreated 等，使得钩子与相关逻辑更紧密地结合。

总结
选项式 API：适合小型项目，简单易懂，但在大型应用中可能变得难以管理。
组合式 API：适合复杂和大型应用，提供更好的逻辑组织和复用能力，更强的 TypeScript 支持。
