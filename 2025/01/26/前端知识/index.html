## 1. axios
## >Axios 是一个基于 Promise 的 HTTP 客户端，用于浏览器和 Node.js。它可以用于发送 HTTP 请求，并支持多种功能。以下是 Axios 的一些基本概念和用法： ### 基本用法 ### 发送 GET 请求 >axios.get('https://api.example.com/data') .then(response => { console.log(response.data); // 处理成功响应 }) .catch(error => { console.error('Error:', error); // 处理错误 });

发送 POST 请求

axios.post(‘https://api.example.com/data‘, {
name: ‘John’,
age: 30
})
.then(response => {
console.log(response.data); // 处理成功响应
})
.catch(error => {
console.error(‘Error:’, error); // 处理错误
});

请求配置

你可以在发送请求时提供配置选项，例如 headers、params 等。

axios.get(‘https://api.example.com/data‘, {
headers: {
‘Authorization’: ‘Bearer your_token’
},
params: {
id: 123
}
})
.then(response => {
console.log(response.data);
})
.catch(error => {
console.error(‘Error:’, error);
});

响应拦截器和请求拦截器

你可以使用拦截器来处理请求和响应。

请求拦截器

axios.interceptors.request.use(config => {
// 在发送请求之前做些什么
console.log(‘Request made with ‘, config);
return config;
}, error => {
// 处理请求错误
return Promise.reject(error);
});

响应拦截器

axios.interceptors.response.use(response => {
// 处理响应数据
return response;
}, error => {
// 处理响应错误
return Promise.reject(error);
});

取消请求

你可以使用 CancelToken 来取消请求。

const CancelToken = axios.CancelToken;
let cancel;
axios.get(‘https://api.example.com/data‘, {
cancelToken: new CancelToken(function executor(c) {
cancel = c;
})
}).then(response => {
console.log(response.data);
}).catch(error => {
if (axios.isCancel(error)) {
console.log(‘Request canceled:’, error.message);
} else {
console.error(‘Error:’, error);
}
});

取消请求

cancel(‘Operation canceled by the user.’);

并行请求

使用 axios.all 发送并行请求。

axios.all([
axios.get(‘https://api.example.com/data1‘),
axios.get(‘https://api.example.com/data2‘)
]).then(axios.spread((data1, data2) => {
console.log(data1.data);
console.log(data2.data);
})).catch(error => {
console.error(‘Error:’, error);
});

总结

Axios 是一个功能强大的 HTTP 客户端，支持各种配置、拦截器和并行请求等功能。它的 API 简洁明了，能够方便地与 RESTful API 进行交互。

2.本地写好的vue项目通过nvm，npm,nodejs生成dist文件的流程

在命令行中使用 cd 命令导航到你的 Vue 项目目录，例如：

>cd "E:\python code\前端\POEMON-BLOG-main\My-Blog-FrontOpen"

确保在项目目录中执行以下命令，以安装项目所需的依赖：

>npm install

现在，你可以运行构建命令来生成 dist 文件夹。通常，这个命令在 package.json 的 scripts 部分中定义，通常是 build。执行以下命令：

>npm run build

部署 dist 文件

接下来，你可以将 dist 文件夹中的内容部署到服务器或静态网站托管服务，如 Netlify、Vercel、GitHub Pages 等。

3. nodejs，npm,nvm的概念

Node.js 是服务器端的 JavaScript 运行时环境。
npm 是 Node.js 的包管理工具，用于管理依赖和发布包。
nvm 是用于管理 Node.js 版本的工具，方便开发者在不同版本之间切换。

4. Webpack概念

Webpack 是一个现代 JavaScript 应用程序的静态模块打包工具。它主要用于将复杂的前端应用程序打包成更小的、可管理的文件，从而优化加载性能。旨在解决现代前端开发中的模块化和资源管理问题，使得开发者能够构建高效、可维护的应用程序。

7. CDN（内容分发网络）

CDN（内容分发网络）是一种通过全球分布的服务器加速内容传输，提高网站加载速度和可靠性的方法

使用 CDN
在网页中使用 CDN 资源非常简单，只需在 HTML 文件中添加相应的链接。例如，要使用 Bootstrap 的 CSS：

link rel=”stylesheet” href=”https://stackpath.bootstrapcdn.com/bootstrap/4.5.2/css/bootstrap.min“

这段代码会从 Bootstrap 的 CDN 服务器加载 CSS 文件，而不是从您的服务器加载。这样可以提高加载速度，并减轻您服务器的压力。
